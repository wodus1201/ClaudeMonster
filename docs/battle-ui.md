# 배틀 화면 드롭다운 UI

메뉴바 위젯을 클릭하면 지금은 평범한 `NSMenu`가 뜬다. 이걸 **포켓몬 2세대
배틀 화면**을 본뜬 완전 커스텀 UI로 바꾸는 작업. 현재 기술 검증(스파이크)까지
끝났고, 실기능은 아직 하나도 붙지 않았다.

- 프로토타입: [`prototypes/battle_spike.swift`](../prototypes/battle_spike.swift)
- 상태: 렌더링·상호작용 검증 완료 / 실기능 미배선 / 본 앱 미통합
- 마지막 갱신: 2026-07-10

## 목표 화면

```
┌──────────────────────────────────────┐
│ 버그:L50                    [버그]   │   상대 = 컴퓨터 버그 (임의 Lv, HP 100%)
│ HP:▁▁▁▁▁▁▁▁                          │
│                                       │
│      [클로드 뒷모습]   클로드:L16     │   나 = 메뉴바와 동일한 Lv/HP
│                        HP:▁▁▁▁ 84/100 │
├──────────────┬───────────────────────┤
│ 무엇을 할까? │ ▶사용량 선택  색상커스텀│   2x2 메뉴
│              │  간결 모드    종료      │
└──────────────┴───────────────────────┘
```

"사용량 선택"을 고르면 배틀 영역은 그대로 두고 **하단 대화상자의 네 항목만**
5시간 / 주간 / Fable / 뒤로로 교체된다.

## 검증된 것 (스파이크에서 확인 완료)

- accessory 앱(`LSUIElement=1`)에서도 패널 내부의 버튼 클릭, 슬라이더 드래그,
  텍스트 입력이 모두 정상 동작한다. 실제 클릭으로 로그를 찍어 확인했다.
- 스프라이트 히트박스(쓰다듬기)와 패널 토글이 공존한다.
- 열림/닫힘 애니메이션 가능 (위에서 내려오며 페이드인).
- 맥북 해상도(노치 포함)에서 패널이 화면 밖으로 잘리지 않는다.

## 설계 결정과 그 이유

**`NSPopover`가 아니라 테두리 없는 `NSPanel`을 쓴다.**
팝오버는 화살표 꼬리와 시스템 반투명 재질, 둥근 모서리가 강제로 붙는다. 이걸
벗기는 공식 방법이 없어서 게임보이 화면과 섞이면 이상해진다. 패널은 배경부터
전부 직접 그린다. 대가로 팝오버가 공짜로 주던 "바깥 클릭 시 닫힘"(`.transient`)을
전역 이벤트 모니터로 직접 구현해야 한다. → 이미 구현됨.

**닫을 때 `panel` 참조를 먼저 비운다.**
애니메이션이 끝날 때까지 참조를 들고 있으면 그 사이 다시 클릭했을 때 토글이
꼬인다. 참조를 먼저 `nil`로 만들고 애니메이션은 지역 변수로 넘긴다.

**클로드 뒷모습은 앞모습(`clawdBase`)에서 파생시킨다.**
새로 그리지 않고 눈을 빼고 등 이음새를 한 칸 넣는다. 앞모습을 수정해도 뒷모습
실루엣이 어긋나지 않게 하려는 것. (본 앱이 아이콘을 스프라이트에서 파생시키는
것과 같은 이유.)

**스프라이트 위치는 좌표를 박지 않고 영역에서 계산한다.**
HP 인디케이터가 차지하지 않은 사각형을 구하고 그 중앙을 기준으로 배치한다.
인디케이터 크기를 바꿔도 스프라이트가 따라온다.

**셰이딩은 대각선 그라데이션 + 경계 디더링.**
GBC는 스프라이트당 3색 + 투명이 한계였다. 그 제약(하이라이트/기본/그림자 3톤 +
검은 외곽선)을 지키되, 각 픽셀의 밝기를 "행 안에서의 가로 위치 + 전체에서의
세로 위치"를 섞은 대각선 그라데이션으로 정한다.

> 폐기한 방법: 열 단위 체커보드 디더링. 세로 줄무늬가 생겨 레트로가 아니라
> 얼룩처럼 보였다. 디더링은 톤이 갈리는 **경계 한 칸에서만** 써야 한다.

**배틀 영역을 먼저 그리고 대화상자가 그 위를 덮는다.**
그래야 클로드의 하반신이 자연스럽게 잘린다(원작 구도).

## 인디케이터 디자인 (원작 대조 결과)

여러 번 틀렸던 부분이라 확정된 규칙을 남긴다.

- HP 게이지는 **얇은 직사각형**. 캡슐/둥근 모서리 아님.
- 이름이 레벨(`:Ln`)보다 폰트가 크다. 같은 베이스라인에 두 크기를 얹는다.
  (NeoDunggeunmo는 단일 weight라 굵기 차이는 크기로만 표현 가능하다.)
- 프레임 = **굵은 세로 띠 + 얇은 아래 선**, 띠가 선보다 약 5배 두껍다.
  아래 선의 한쪽 끝에는 상하 반으로 잘린 화살촉이 붙는다.
- 상대: 왼쪽 세로 띠, 화살촉은 아래 선 오른쪽 끝. 게이지 최우측에 두꺼운
  검정 마감 띠(트랙이 끝까지 가지 않고 멈춘다).
- 나: 좌우반전. 오른쪽 세로 띠에 게이지가 직각으로 이어진다. 화살촉은 왼쪽 끝.
  큰 HP 숫자, 그리고 **컨테이너 없는** 경험치 바(파란 채움만)가 아래 선 살짝
  위에 놓이며 **우→좌**로 채워진다.
- 대화상자와 메뉴 컨테이너 모두 더블라인(3pt 외곽 + 1pt 내곽).
- 배경은 메뉴바 위젯의 알약 배경과 같은 `NSColor(white: 0.90, alpha: 1)`.

## 반복해서 밟은 함정

**플레이어 인디케이터는 상대보다 높아야 한다.** 이름/게이지/숫자/경험치 네
줄이 들어가는데 상대와 같은 높이를 주면 숫자가 게이지를 침범한다. 눈으로
넘기지 말고 폰트 메트릭으로 각 요소의 y 구간을 계산해 겹침을 확인할 것.

**받침선은 "마지막 요소 바로 아래"에 붙인다.** 박스 바닥을 기준으로 잡으면
아래가 빈 상대 인디케이터에서 선이 허공에 뜬다.

**안티앨리어싱을 끈 뒤 그래픽 상태를 복원할 것.** 안 하면 이후에 그려지는
대화상자 곡선까지 전부 계단진다.

## 남은 일

1. **실기능 배선** — 사용량 전환(`selectedKind`), 간결 모드, 종료.
   "색상 커스텀"은 아직 기능이 정해지지 않은 자리.
2. **본 앱 통합** — `buttonClicked()`의 "메뉴를 임시로 붙였다 떼는" 트릭을
   패널 토글로 교체. 스프라이트 히트박스는 그대로 유지.
3. **세 가지 상태 대응** — `rebuildMenu()`가 만드는 사용량 / 조는 중(429) /
   오류(로그인 없음) 세 메뉴를 배틀 화면의 세 상태로 옮겨야 한다. HP를 모를 때
   (`--/100`) 화면을 어떻게 그릴지 정해지지 않았다.
4. **탈출구 유지** — 종료를 배틀 메뉴에만 두면 화면이 깨졌을 때 앱을 끌 방법이
   없다. 우클릭에는 최소 `NSMenu`(종료, 업데이트 확인)를 남기는 것을 권한다.
5. 디자인 미세 조정 (띠 두께, 화살촉 크기, 버그 스프라이트).

## 릴리즈 시 주의

`VERSION`은 아직 `1.2`이고 태그 `v1.2`가 이미 존재한다. 배포하려면 `1.3`으로
올려야 한다. 배틀 UI가 실기능까지 붙은 다음에 올리는 것을 전제로 두었다.
