# 배틀 화면 드롭다운 UI

메뉴바 위젯을 **좌클릭**하면 포켓몬 배틀 화면을 본뜬 패널이 내려온다.
**우클릭**은 기존 `NSMenu`를 그대로 띄운다.

- 코드: [`src/main.swift`](../src/main.swift) 의 `// MARK: - Battle screen` 이후
- 상태: 통합 완료. 메뉴 상호작용 동작. 색상 커스텀만 미구현.
- 마지막 갱신: 2026-07-10

```bash
# 창 없이 세 페이지를 PNG로 뽑아 디자인 확인
CLAUDEMONSTER_BATTLE=/tmp/b.png ./build/ClaudeMonster.app/Contents/MacOS/ClaudeMonster

# 네트워크 없이 실제로 클릭해 보기
CLAUDEMONSTER_MOCK=1 ./build/ClaudeMonster.app/Contents/MacOS/ClaudeMonster
```

## 화면 구성

상대는 **고오스**(1세대 참조), 나는 **클로드 뒷모습**. 클로드의 Lv과 HP는
메뉴바 위젯이 보여주는 값과 동일하다(추적 중인 한도의 `used%`).

하단 대화상자는 세 페이지를 오간다. **배틀 장면은 그대로 두고 대화상자만
교체**된다.

| 페이지 | 좌상 | 우상 | 좌하 | 우하 |
|---|---|---|---|---|
| 루트 | 사용량 선택 | 색상 커스텀 *(미구현·비활성)* | 기능 더보기 | 종료하기 |
| 사용량 | 5시간 | 주간 | Fable | 뒤로 |
| 기능 더보기 | 간결 모드 ✓ | 버전 확인 | 새로고침 | 뒤로 |

사용량 페이지의 항목은 **계정의 실제 한도 목록에서 생성**된다. 하드코딩하면
Fable 한도가 없는 계정에서 유령 항목이 뜬다. 커서(▶)는 현재 추적 중인
항목에서 시작한다.

## 설계 결정과 그 이유

**`NSPopover`가 아니라 테두리 없는 `NSPanel`.**
팝오버는 화살표 꼬리와 시스템 반투명 재질을 벗길 수 없어 픽셀아트와 섞이면
이상해진다. 대가로 "바깥 클릭 시 닫힘"(`.transient`)을 전역 이벤트 모니터로
직접 구현했다.

**우클릭 메뉴를 남긴다.**
배틀 화면이 깨지면 앱을 끌 방법이 사라진다(Dock 아이콘이 없다). 데이터가
없을 때(로그인 전, 429 백오프 중)도 그릴 HP가 없으므로 메뉴로 폴백한다.
그 메뉴가 "로그인이 필요합니다" 같은 **이유까지 설명**하므로 빈 배틀 화면보다
낫다.

**상태 변경은 우클릭 메뉴와 같은 코드를 공유한다.**
`selectLimit(kind:)`, `toggleCompact()`를 양쪽이 함께 호출한다. 배틀 화면이
따로 상태를 만지면 위젯과 어긋난다. 액션 후 `refreshBattleView()`로 패널을
델리게이트 상태에서 다시 읽는다.

**새로고침은 즉시 다시 그리면 안 된다.**
`refreshNow()`는 비동기 fetch다. 바로 `refreshBattleView()`를 부르면 옛
숫자를 그린다. 응답이 반영될 시간을 두고 갱신한다.

**버전 확인은 패널을 먼저 닫는다.** 모달 알림이라 패널이 뒤에 매달린다.

**애니메이션 타이머는 `viewDidMoveToWindow`에 묶는다.**
패널이 닫히면 타이머도 죽어야 한다. 안 그러면 타이머가 뷰를 붙잡아 누수가 된다.

**클로드 뒷모습은 `clawdBase`에서 파생.**
앞모습을 고치면 뒷모습과 앱 아이콘이 함께 따라온다. 등 이음새는 넣지 않는다 —
실제 2세대 백스프라이트에 그런 선이 없고, 얼굴만 없애면 이미 뒷모습으로 읽힌다.

## 스프라이트를 만질 때

`battleShaded()`는 **`B`(몸통)만 재채색**한다. 눈·이빨·코어처럼 자기 색을
지켜야 하는 픽셀은 별도 문자(`W`/`P`/`T`/`C`)로 찍어야 한다. `B` 위에 얹으면
그라데이션이 덮어써서 뭉개진다.

셰이딩은 대각선 그라데이션 + **경계에서만** 체커보드 디더링.

> 폐기: 열 단위 디더링. 세로 줄무늬가 생겨 얼룩처럼 보였다.

편집 후에는 **행 수와 폭이 균일한지 반드시 확인**한다. 어긋나도 컴파일은
통과하고 렌더만 조용히 깨진다.

```bash
python3 - <<'PY'
import re
src = open('src/main.swift').read()
rows = re.findall(r'"([^"]*)"', re.search(r'let bugBase: \[String\] = \[(.*?)\n\]', src, re.S).group(1))
print(len(rows), {len(r) for r in rows})   # 20 {22} 여야 한다
PY
```

고오스를 그리며 배운 것: 코어를 가로로 긴 사각형으로 두면 구체로 안 보인다.
이빨은 흰 픽셀을 띄엄띄엄 찍으면 점선처럼 읽히고, **흰 띠 한 줄 + 그 아래로
물려 내려오는 이빨**이라야 톱니가 된다. 눈은 모서리를 깎아야 둥글게 보인다.

## 애니메이션

클로드는 **4px 사인 아치**. 결과를 반올림해 픽셀 그리드에 고정한다. 반픽셀에
착지하면 스프라이트가 흐려진다. (메뉴바 위젯이 1px인 건 메뉴바 높이가 22px뿐이라서.)

고오스는 **목표점을 향해 보간**한다. 매 프레임 난수를 더하면 부유가 아니라
진동으로 보이고, 목표가 현재 위치와 가까우면 경련처럼 보인다 — 그래서 충분히
먼 목표만 뽑는다. 드리프트 범위(±16/±13px)는 인디케이터와 화면 경계를 침범하지
않는 선에서 계산했다.

## 레이아웃의 함정

**메뉴 셀 폭에 여유가 거의 없다.** 이 값들이 서로 물려 있다:

```
BATTLE_W(480) · menuRatio(0.60) · cellPadX(12) · cursorW(18) · itemFontSize(16)
```

가장 긴 항목("간결 모드 ✓", 89.6pt)이 가용 폭(90.8pt)에 **1.2pt 남기고**
들어간다. 왼쪽 메시지("어떤 걸 해볼까?")도 함께 검사해야 한다. 패딩이나 커서
간격만 늘리면 글자가 잘린다. 항목 이름을 바꿀 때도 폭을 다시 재라.

**플레이어 인디케이터는 64pt가 바닥이다.** 이름·게이지는 위에서, 숫자·경험치는
아래에서 쌓이므로 그보다 낮추면 HP 숫자가 경험치 바를 침범한다.

**받침선은 "마지막 요소 바로 아래"에 붙인다.** 박스 바닥 기준으로 잡으면
아래가 빈 상대 인디케이터에서 선이 허공에 뜬다.

**안티앨리어싱을 끈 뒤 그래픽 상태를 복원할 것.** 안 하면 이후에 그려지는
대화상자 곡선까지 계단진다.

**`cacheDisplay`는 둥근 패널 바깥을 투명하게 남긴다.** PNG 뷰어가 흰색으로
칠해 렌더링 버그처럼 보이므로, 덤프는 어두운 배경을 먼저 깐다.
